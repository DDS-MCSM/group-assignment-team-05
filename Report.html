Creacin de modelo estadstico para la deteccin de malware en APKs.

Requerimientos

Para este proyecto, necesitaremos un conjunto de APKs clasificadas segn sean malware y benignas. En nuestro caso, usaremos un conjunto publicado por el Canadian Institute for Cybersecurity, el cual realiz en 2017 tambin un estudio para intentar determinar si una app es malware o no en base al trfico de red que generaba. El estudio recoge los datos de 10.854 apps, de las cuales 4.354 fueron detectadas como malware y 6500 como benignas. El origen de las apps proviene de diferentes fuentes entre las cuales 6.000 tienen el origen en la play store Oficial. A partir del anlisis de las apps se clasificaron en 4 categoras: Adware, Rasonmware, Scareware, SMS Malware.

Objetivos

Nuestro objetivo principal es intentar crear un modelo que nos permita determinar si una aplicacin es malware o no usando nicamente los datos que se pueden extraer utilizando la herramienta apkanalyzer del SDK oficial de Android. Para ello, testearemos dos modelos de clasificacin supervisada: KNN y SVM.

Adquisicin de datos

Como hemos dicho, hemos utilizado las APKs a partir del proyecto publicado por el Canadian Institute for Cybersecurity. Para extraer la informacin correspondiente, realizamos un script en python, el cual generaba una tabla en CSV para que pudiesemos tratar los datos extraidos a partir de la herramienta apkanalyzer. Dicha herramienta nos permiti crear un conjunto de datos de cada apk con las siguientes columnas:

  • name: El nombre de la aplicacin
  • type: El tipo de la aplicacin, es decir, si es benigna o algn tipo de malware
  • apk_size: El tamao del archivo apk
  • n_files: El nmero de archivos contenidos en el apk
  • n_permissions: El nmero de permisos

Adems de estos datos bsicos, mediante el apkanalyzer tambin podemos obtener un rbol con las referencias del cdigo de la aplicacin, lo que nos permiti obtener columnas con el nmero de paquetes, clases, mtodos y campos definidos y referenciados, as como el tamao de stos.

Las columnas correspondientes son: n_defined_packages, n_referenced_packages, n_defined_classes, n_referenced_classes, n_defined_methods, n_referenced_methods, n_defined_fields, n_referenced_fields.

Finalmente, tambien tenemos el campo max_depth que indica la profundidad mxima de un objeto en el rbol de cdigo, lo que nos puede indicar en cierta manera la complejidad de la aplicacin.

Creacin de los modelos

Como hemos comentado antes, hemos intentado encontrar un modelo basado en KNN y SVM que nos permita determinar si una aplicacin es malware o no. Basndonos en varias pruebas, hemos visto que el modelo que mejor se ajusta a nuestros datos es uno que tenga en cuenta los siguientes datos:

  • El tamao de la aplicacin apk_size
  • El nmero de mtodos definidos n_defined_methods
  • Una nueva variable max_depth_le_4, que cuyo valor es 1 o 0 en funcin de si la variable max_depth es menor o igual o mayor a 4.

Para la creacin de los modelos hay que tener en cuenta que decidimos normalizar todos los datos, y que por falta de tiempo, no pudimos experimentar asignando pesos diferentes a cada variable, por lo que todas tienen el mismo peso.
